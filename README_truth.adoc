Gradle plugin published under the Apache 2.0 license on the link:https://plugins.gradle.org[Gradle Plugin Portal].

It automatically generates GitHub-compatible `README.adoc` files from `README_plantuml.adoc` source files containing native PlantUML blocks.

== 1. Problem solved

GitHub does not natively render PlantUML syntax in AsciiDoc files.
A `[plantuml]` block displayed as-is on github.com is not rendered as an image.

This plugin automates the transformation via GitHub Actions:

[plantuml, problem-solution, png]
----
@startuml
skinparam componentStyle rectangle

package "What you write" {
  [README_plantuml.adoc\n(native plantuml blocks)] as src #LightGreen
}

package "What GitHub displays" {
  [README.adoc\n(image::...png references)] as out #LightBlue
}

package "What the visitor sees" {
  [Diagrams rendered\nin the browser] as web #LightYellow
}

src --> out : plugin generates\n(GitHub Actions)
out --> web : GitHub renders\nnatively
@enduml
----

== 2. Philosophy: single source of truth

`README_plantuml.adoc` is the **only file you edit**.
`README.adoc` is a generated artifact — never edited by hand.

[plantuml, source-of-truth, png]
----
@startuml
skinparam defaultTextAlignment center

rectangle "You edit" #LightGreen {
  file "README_plantuml.adoc" as src
  file "README_plantuml_fr.adoc" as srcFr
}

rectangle "CI generates" #LightBlue {
  file "README.adoc" as out
  file "README_fr.adoc" as outFr
  folder ".github/workflows/readmes/images/en/" as imgEn
  folder ".github/workflows/readmes/images/fr/" as imgFr
}

src --> out
src --> imgEn
srcFr --> outFr
srcFr --> imgFr
@enduml
----

== 3. How it works

=== Overview

[plantuml, overview-sequence, png]
----
@startuml
actor Developer
participant "GitHub" as GH
participant "GitHub Actions" as CI
participant "processReadme\n(Gradle Task)" as proc
participant "commitGeneratedReadme\n(Gradle Task)" as commit
participant "JGit" as jgit

Developer -> GH : git push README_plantuml*.adoc
GH -> CI : triggers workflow
CI -> proc : ./gradlew commitGeneratedReadme
proc -> proc : detects [plantuml,name,...] blocks
proc -> proc : generates .github/workflows/readmes/\nimages/{lang}/name.png
proc -> proc : rewrites README.adoc\nwith image::...png[]
proc -> commit : dependsOn resolved
commit -> jgit : git add + commit + push
jgit -> GH : README.adoc + images committed
GH --> Developer : README visible ✓
@enduml
----

=== Activity diagram

[plantuml, activity, png]
----
@startuml
start

:Read readme-plantuml.yml\n(injected from CI secret);

:Scan README_plantuml*.adoc\nin sourceDir;

if (Files found?) then (no)
  :Warning — no source file;
  stop
else (yes)
  :Detect language\nfrom _xx suffix;
endif

repeat
  :Extract [plantuml, name, ...] blocks;

  if (Blocks found?) then (yes)
    repeat
      :Generate PNG\nbuild/img/{lang}/name.png;
      :Copy to\n.github/workflows/readmes/images/{lang}/;
      :Replace block with\nimage::.github/.../name.png[name];
    repeat while (more blocks?)
  endif

  :Write README{_lang}.adoc;
repeat while (more source files?)

:git add README*.adoc\n+ .github/workflows/readmes/images/;
:git commit + push\nvia JGit (no git binary required);

stop
@enduml
----

== 6. Configuration

Configuration is externalized in `readme-plantuml.yml`.
This file **never** contains the token in plain text in the repository.
Its full content (token included) is stored in the GitHub secret `README_GRADLE_PLUGIN`.

=== `readme-plantuml.yml` template

[source,yaml]
----
source:
  dir: "."                    # <1>
  defaultLang: "en"           # <2>

output:
  imgDir: ".github/workflows/readmes/images"  # <3>

git:
  userName: "github-actions[bot]"
  userEmail: "github-actions[bot]@users.noreply.github.com"
  commitMessage: "chore: generate readme [skip ci]"
  token: "<YOUR_GITHUB_PAT>"  # <4>
  watchedBranches:
    - "main"
    - "master"
----
<1> Directory containing the `README_plantuml*.adoc` files
<2> Default language when no `_xx` suffix is present in the filename
<3> Generated images folder — versioned in git
<4> Replaced by the real PAT only inside the GitHub secret

=== Secret injection in CI

[source,yaml]
----
- name: Inject plugin config
  run: echo "${{ secrets.README_GRADLE_PLUGIN }}" > readme-truth.yml

- name: Generate README and commit via JGit
  run: ./gradlew -q -s commitGeneratedReadme --no-daemon
----

== 8. Full GitHub Actions workflow

[source,yaml]
----
name: Generate README from PlantUML sources

on:
  push:
    branches:
      - main
      - master
    paths:
      - "README_plantuml*.adoc"

  workflow_dispatch:

jobs:
  generate-readme:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
          cache: gradle

      - name: Grant execute permission for gradlew
        run: chmod +x gradlew

      - name: Inject plugin config
        run: echo "${{ secrets.README_GRADLE_PLUGIN }}" > readme-truth.yml

      - name: Generate README and commit via JGit
        run: ./gradlew -q -s commitGeneratedReadme --no-daemon
----
